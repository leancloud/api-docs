Overview

Represents a Role on the LeanCloud server. AVRoles represent groupings of AVUsers for the purposes of granting permissions (e.g. specifying a AVACL for a AVObject). Roles are specified by their sets of child users and child roles, all of which are granted any permissions that the parent role has.

Roles must have a name (which cannot be changed after creation of the role), and must specify an ACL.

  name

Gets or sets the name for a role. This value must be set before the role has been saved to the server, and cannot be set once the role has been saved.

A role’s name can only contain alphanumeric characters, _, -, and spaces.
